    Stories:
        - All notifications are performed by notification service
            When some notification is triggered in Kairos, Kairos sends the request for sendig the notification
            to the notification service that can handle email with the future plans for SMS and posible Slack or in-app
            notifications.

            Kairos needs to be able to send requests for sending notification via HTTP or message queue in which Kairos is the producer
            and notification service is subscriber.
        - Sending emails
            When notification service receives the request for sending and email, that request is placed in email queue. Email handeler
            reads the requests from the email queue and sends the emails. If an email failed to send, that request is placed back into email
            queue. If the same email failed to send, notification service will send an error message via SMS, Slack or some other means to the
            Kairos admins.
